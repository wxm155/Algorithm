## 贪心算法的定义

```txt
贪心算法是指在对问题求解时，总是做出在当前看来是最好的选择。不从整体最优上加以考虑，只做出在某种意义上的局部最优解。贪心算法不是对所有问题都能得到整体最优解，关键是贪心策略的选择，选择的贪心策略必须具备无后效性，即某个状态以前的过程不会影响以后的状态，只与当前状态有关。
```

## 贪心算法解题步骤

```txt
1、建立数学模型来描述问题；

2、把求解的问题分成若干个子问题；

3、对每一子问题求解，得到子问题的局部最优解；

4、把子问题的局部最优解合成原来问题的一个解。
```

## 贪心算法和动态规划的区别

```txt
动态规划和贪心算法它们之间的相似之处，最优解问题大部分都可以拆分成一个个的子问题，把解空间的遍历视为对子问题树的遍历，对整个树的遍历求出最优解，大部分情况下这是不可行的，贪心算法和动态规划本质上是对子问题树的一种剪枝，两种算法要求问题都具有的一个性质就是子问题最优性(组成最优解的每一个子问题的解，对于这个子问题本身也是最优解)，动态规划方法代表了这一类问题的一般解法，我们自底向上构造子问题的解，对每一个字数的根，求出下面每一个叶子的值，以其中的最优值作为自身的值。而贪心算法是动态规划的一个特例，每一个子树的根的值不取决于下面叶子的值，而只取决于当前问题的状况，即不需要知道一个节点所有子树的情况，就可以求出这个节点的值，由于贪心算法这个特性，它对解空间树的遍历不需要自底向上，而只需要自根开始，选择最优的路，一直走到底就可以了。
```
